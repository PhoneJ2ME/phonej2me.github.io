Introduction

This guide contains information for building and running the phoneME™ Feature software, an optimized Java™ 2 Platform, Micro Edition (J2ME™) stack based on Milestone Release 2 (MR2) of the phoneME Feature software project.

This guide allows you to take the following general steps:

  • Set up and configure the build environment for your build platforms
  • Build a default version of the JavaCall™ porting layer (for Win32 on i386 only), PCSL libraries, CLDC, and phoneME Feature software binaries.
  • Run the phoneME Feature software on your default platform.
  • Find additional information about phoneME Feature software.

NOTE: This document is part of the MR2 release of phoneME Feature software. The contents are subject to change at any time, based on community input and ongoing development.

Build Requirements

When building the phoneME Feature software, the machine where you setup your build environment and carry out the build instructions is called your build platform. The build platform is the machine where source code is installed, the compilers are run, and executable code is built. This executable code is then installed and run on a target platform.

The phoneME Feature software supports the following build platforms:

  • Linux on i386
  • Win32 on i386

The phoneME Feature software supports the following target platforms:

  • Linux on ARM (Texas Instruments P2SAMPLE64-V6 board)
  • Linux on i386 (emulation mode only)
  • JavaCall API on i386 (emulation mode only)

Requirements for Building on a Linux on i386 Platform

To properly build both Linux on i386 and Linux on ARM target platforms, a Linux on i386 build platform must meet the following requirements:

  • Ubuntu Linux version 6.06
  • Java 2 Platform, Standard Edition (J2SE™) Development Kit (JDK™) version 1.4.2
  • GNU Make version 3.79.1 or later
  • GNU Cross Compiler (GCC) 3.4.6 or later

For the Linux on ARM target platform, the following software is also required: n

  • MontaVista Linux Consumer Electronics Edition 3.1

The following software is optional:

  • Doxygen version 1.4.1 (used to build PCSL API documentation)

For more information on acquiring and installing Linux on i386 build platform tools, see Setting Up the Linux on i386 Build Environment.

Requirements for Building on a Win32 on i386 Platform

A Win32 on i386 target platform must meet the following requirements:

  • Windows 2000 or Windows XP
  • JDK version 1.4.2
  • Microsoft Visual C++ (MSVC++) Professional Version 6.0 (with Microsoft Processor Pack 5, version 6.15 or later, and an up-to-date Service Pack) or Visual Studio .NET 2003
  • Cyg4Me version 1.1 (a version of Cygwin for building Java ME technology. Cyg4Me contains GNU Make 3.79.1.)

The following software is optional:

  • Doxygen version 1.4.1 (used to build PCSL API documentation)

For more information on acquiring and installing Win32 on i386 build platform tools, see Setting Up the Win32 on i386 Build Environment.
